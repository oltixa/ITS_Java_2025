# Контрольні питання

## 1. Чим компіляція відрізняється від інтерпретації? Які переваги кожної з них?

### Компіляція
**Визначення**: Процес перетворення початкового коду в машинний код (або проміжний код) перед виконанням програми.

**Переваги**:
- Швидше виконання програми
- Виявлення помилок на етапі компіляції
- Оптимізація коду компілятором
- Незалежність від наявності компілятора на цільовій машині

### Інтерпретація
**Визначення**: Процес виконання програми рядок за рядком у реальному часі без попереднього перетворення в машинний код.

**Переваги**:
- Швидша розробка (немає етапу компіляції)
- Інтерактивність і можливість динамічних змін
- Кросплатформність
- Простіша налагодження

## 2. Що таке початковий код, байткод, машинний код? Яка між ними різниця? Що таке JIT-компіляція?

### Початковий код (Source Code)
Код, написаний програмістом на мові програмування високого рівня (наприклад, Java). Читається людиною, але не може виконуватися комп'ютером безпосередньо.

### Байткод (Bytecode)
Проміжний код, в який компілюється початковий код Java. Це платформно-незалежний код, який виконується віртуальною машиною Java (JVM).

### Машинний код (Machine Code)
Низькорівневі інструкції, які безпосередньо виконуються процесором. Специфічний для конкретної архітектури процесора.

### JIT-компіляція (Just-In-Time)
Технологія компіляції байткоду в машинний код під час виконання програми. Дозволяє оптимізувати код на основі реальних умов виконання.

## 3. Що таке віртуальна машина?

**Віртуальна машина (VM)** - це програмне середовище, яке емулює роботу реального комп'ютера. У контексті Java:

- **JVM (Java Virtual Machine)** виконує байткод Java
- Забезпечує абстракцію від операційної системи
- Управляє пам'яттю (garbage collection)
- Забезпечує безпеку виконання коду
- Дозволяє досягти принципу "написав один раз - виконуй скрізь"

## 4. Що таке кросплатформність? Чи є кросплатформними застосунки на мові Java? Чи є кросплатформною віртуальна машина Java?

### Кросплатформність
Здатність програмного забезпечення працювати на різних операційних системах і апаратних платформах без модифікацій.

### Застосунки Java
**Так**, застосунки Java є кросплатформними:
- Компілюються в байткод один раз
- Можуть виконуватися на будь-якій платформі з JVM
- Принцип "Write Once, Run Anywhere" (WORA)

### Віртуальна машина Java
**Ні**, JVM не є кросплатформною:
- Кожна операційна система має свою реалізацію JVM
- JVM адаптована під специфіку конкретної ОС
- Існують різні реалізації JVM для різних платформ

## 5. Пояснення слів у коді HelloWorld

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

### Пояснення кожного слова:

- **class** - ключове слово для оголошення класу
- **public** - модифікатор доступу, означає публічний доступ
- **static** - означає, що метод належить класу, а не об'єкту
- **void** - тип повертаного значення (нічого не повертає)
- **String** - тип даних для рядків
- **System** - стандартний клас Java для системних операцій
- **out** - статичне поле класу System для виводу
- **println** - метод для виводу рядка з переходом на новий рядок

## 6. Зміна регістру літер

Якщо змінити регістр, наприклад, написати `CLASS` замість `class`:

**Результат**: Компілятор видасть помилку!

**Причина**: Java є регістрозалежною мовою (case-sensitive). Всі ключові слова мають точно визначений регістр літер. `class` і `CLASS` - це різні токени для компілятора.

## 7. Встановлення кількох версій Java

### Чи можна встановити кілька версій?
**Так**, можна встановити кілька версій Java одночасно.

### Як це зробити:
1. Встановити різні версії JDK у різні директорії
2. Налаштувати змінну середовища `JAVA_HOME`
3. Використовувати інструменти управління версіями:
   - **Windows**: chocolatey, scoop
   - **macOS**: Homebrew, jenv
   - **Linux**: update-alternatives, SDKMAN!

### Для чого це потрібно:
- Підтримка legacy-проектів на старих версіях
- Тестування додатків на різних версіях Java
- Використання нових фічей у нових проектах
- Розробка під різні середовища виконання
- Сумісність з різними фреймворками і бібліотеками
